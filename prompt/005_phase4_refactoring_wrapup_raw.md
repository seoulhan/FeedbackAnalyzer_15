# [Phase 4] 리팩토링 진행 - 네이밍 및 비대 함수 개선 요청
당신은 클린 코드 구현 전문가입니다.  @docs/analysis.md 의 내용을 바탕으로 리팩토링을 진행합니다.
대표적으로, `TextAnalyzer` 및 `Filters` 소스코드의 가독성 향상과 하드코딩 제거 작업을 위하여 다음의 요구사항을 진행합니다.

[대표 개선 요구사항]
1. 함수명 도메인 용어화 및 명확화:
 - `sent()` -> `analyzeSentiment()`
 - `kw()` -> `analyzeKeywords()`
 - `fil()` -> `filter()`
 - `fil_data` (전역 변수) -> `FilterResult` (의미 명확한 변수 혹은 구조체 반환 형태로 대체)
 - `getOldDataFromSession()` -> `getCurrentFeedbacks()`
2. 매직 넘버 상수화:
 - 가중치 비교 기준값, 루프 제어 상수 등 코드 곳곳에 존재하는 의미 불분명한 하드코딩 숫자들을 `Constants` 영역으로 추출하거나 파일 내 상단에 `const` 기법으로 의미 있는 도메인 용어로 명명하여 상수화해 주세요.
3. 20줄 이상 비대 메서드 분리:
 - `TextAnalyzer` 내에 20줄이 넘어가는 긴 제어 함수들을 분석하여 단일 책임(SRP)에 부합하는 하위 프라이빗(private) 헬퍼 메서드들로 추출해 주세요.

작업 도중 기존에 연동했던 Google Test에 컴파일 에러가 나지 않도록 전체 파일의 호출 구조가 유기적으로 호환되게 주의하여 정돈해 주세요.

[검증 루프]
- 수정마다 cmake 로 빌드하여 정상적인 동작이 보장되는지 확인하고, 실패할 경우 되돌려서 다시 검토하십시오.
