# [Phase 5 & 6] 리팩토링 진행 - SRP 기반 컨트롤러 및 데이터 서비스 클래스 분리
당신은 클린 코드 구현 전문가입니다.  @docs/analysis.md 의 내용을 바탕으로 리팩토링을 진행합니다.
현재 `main.cpp` 및 HTTP 컨트롤러 영역은 웹 요청 처리(HTTP)와 내부 감성 분석 데이터 가공/조작 로직이 사납게 엉켜 있는 SRP(단일 책임 원칙) 위반 상태입니다. 코드를 다음 지침을 기반으로 개선합니다.

[리팩토링 지침]
1. `FeedbackController`의 역할 축소:
 - 외부 HTTP 라우팅 및 텍스트 데이터의 입출력 JSON 매핑 가공 처리만 담당하도록 책임을 경계(Boundary) 레이어로 통제합니다.
2. `FeedbackService` 클래스 분리 추출:
 - 비즈니스 컨트롤러 로직(감성 분석 실행, 키워드 맵 결합 필터링, 세션 연동 연산 등)을 전담하는 순수 서비스 클래스를 별도로 추출해 주세요.
3. 이로 인해 변경되는 전체 의존 구조에 대한 세부 명세와 리팩토링 전후 아키텍처 비포/애프터 비교 분석표를 Markdown 형식으로 `refactoring.md` 파일에 저장해 주세요.
